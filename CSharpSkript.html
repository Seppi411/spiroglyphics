Was ist Objektorientierung?

Die Objektorientierung ist ein Modellierungsansatz und Programmierparadigma, bei dem Objekte und deren Wechselwirkungen aus der Realität in ein Modell und später in ein Programm überführt werden. Durch die Realitätsnähe des Modells und des Programms entstehen leichter änderbare und wartbare IT-Systeme.

Ziele der Objektorientierung

Die Objektorientierung verfolgt mehrere zentrale Ziele:

  • Modularität: Der Code wird in kleinere, unabhängige Einheiten (Klassen und Objekte) aufgeteilt, was die Entwicklung und Wartung erleichtert.
  • Wiederverwendbarkeit: Durch die Struktur in Klassen lassen sich Codebestandteile effizient wiederverwenden.
  • Erweiterbarkeit: Objektorientierte Programme sind so aufgebaut, dass sie leicht um neue Funktionalitäten ergänzt werden können, ohne bestehenden Code zu verändern.
  • Wartbarkeit: Die Strukturierung in Klassen und Objekte macht Programme verständlicher und vereinfacht die Wartung.

Was ist ein Objekt?

Ein Objekt in der Informatik repräsentiert einen digitalen Zwilling eines in der Realität existierenden Objektes. Ein Objekt kann sowohl Attribute (Daten) als auch Methoden (Funktionen) enthalten, die dessen Verhalten definieren.

Was sind Attribute?

Attribute, auch als Eigenschaften bezeichnet, gehören zu einem Objekt und beschreiben dieses näher. Sie stellen die Daten dar, die das Objekt charakterisieren, wie zum Beispiel Farbe, Größe oder andere relevante Merkmale.

Was ist eine Methode?

Methoden beschreiben Dinge, die von Objekten getan werden können. Sie definieren das Verhalten eines Objekts. Zum Beispiel kann ein Objekt, das eine Person repräsentiert, Methoden haben wie trinken, sprechen oder Auskunft geben.

Die drei Phasen der Objektorientierung

Die objektorientierte Programmierung umfasst drei zentrale Phasen:

  • Objektorientierte Analyse (OOA): In dieser Phase werden die Anforderungen an das System erfasst und analysiert. Es werden Objekte identifiziert, die aus der realen Welt stammen, und deren Beziehungen zueinander beschrieben.
  • Objektorientiertes Design (OOD): Hier wird das System entworfen. Die identifizierten Objekte werden in Klassen strukturiert, und die Beziehungen sowie die Interaktionen zwischen diesen Klassen werden festgelegt.
  • Objektorientierte Programmierung (OOP): In dieser Phase wird der tatsächliche Code geschrieben. Die Klassen und Objekte werden programmiert, und die definierten Methoden und Attribute werden implementiert.

Prinzip der Datenkapselung

Das Prinzip der Datenkapselung bezieht sich darauf, dass die internen Zustände eines Objekts vor der Außenwelt verborgen werden. Dadurch werden die Daten eines Objekts geschützt und nur über definierte Methoden (Getter und Setter) zugänglich gemacht. Dies hat mehrere Vorteile:

  • Schutz der Datenintegrität: Durch den kontrollierten Zugriff auf die Daten können unerwünschte Änderungen oder Fehler vermieden werden.
  • Verbesserte Wartbarkeit: Änderungen an den internen Implementierungen eines Objekts wirken sich nicht auf den Code aus, der das Objekt verwendet, solange die Schnittstelle unverändert bleibt.
  • Reduzierte Komplexität: Der Benutzer des Objekts muss sich nicht um die internen Details kümmern und kann sich auf die bereitgestellten Methoden konzentrieren.

Was ist eine Klasse und was ist ein Objekt?

In der objektorientierten Programmierung ist eine Klasse eine Vorlage oder ein Bauplan, der beschreibt, wie Objekte erzeugt werden. Eine Klasse definiert die Attribute (Daten) und Methoden (Funktionen), die die Objekte besitzen.

Ein Objekt hingegen ist eine Instanz einer Klasse. Es stellt ein konkretes Beispiel dar, das auf der in der Klasse definierten Struktur basiert. Jedes Objekt hat eigene Werte für die Attribute, die in der Klasse definiert sind. Beispielsweise könnte die Klasse „Auto“ verschiedene Objekte repräsentieren, wie ein rotes Auto und ein blaues Auto, die unterschiedliche Werte für das Attribut „farbe“ haben.

Klassenmodifizierer in C#

In C# gibt es verschiedene Modifizierer, die das Verhalten von Klassen und deren Mitgliedern steuern. Hier sind die wichtigsten Modifizierer:

1. Objekt-Klassen

Eine öffentliche Klasse, die keinen Modifizierer hat:


    public class CMyClass { }
  • Eigenschaften:
    • Öffentliche Klasse: public (ohne Angabe ist die Klasse automatisch private!)
    • Standard Sichtbarkeit der Attribute: private
    • Kann Implementierungen enthalten und statische Member haben.
    • Kann vererbt werden.
    • Beliebig viele Objekte können instanziiert werden.

2. Statische Klasse

Eine Klasse, die mit dem Modifizierer static deklariert wird:


    public static class CMyClass { }
  • Eigenschaften:
    • Alle Member müssen statisch sein.
    • Zugriff über den Klassennamen: CMyClass.Output()
    • Statische Klassen können nicht vererbt werden.

3. Versiegelte Klassen

Eine Klasse, die nicht vererbt werden kann:


    public sealed class CMyClass { }
  • Eigenschaften: Kann nicht vererbt werden.

4. Abstrakte Klassen

Eine Klasse, die mit dem Modifizierer abstract deklariert wird:


    public abstract class CMyClass { }
  • Eigenschaften:
    • Abstrakte Klassen müssen vererbt werden.
    • Sie können abstrakte Member enthalten, die in abgeleiteten Klassen implementiert werden müssen.

Was ist UML?

UML (Unified Modeling Language) ist eine standardisierte Modellierungssprache, die verwendet wird, um das Design und die Architektur von Software-Systemen zu visualisieren, zu spezifizieren, zu konstruieren und zu dokumentieren. UML bietet eine Vielzahl von Diagrammen, um unterschiedliche Aspekte eines Systems darzustellen, und ermöglicht es Entwicklern, ihre Entwürfe klar und verständlich zu kommunizieren.

Typen von UML-Diagrammen

UML-Diagramme werden in zwei Hauptkategorien unterteilt: Strukturdiagramme und Verhaltensdiagramme.

1. Strukturdiagramme

Strukturdiagramme zeigen die statische Struktur eines Systems und die Beziehungen zwischen seinen Elementen. Einige Beispiele für Strukturdiagramme sind:

  • Klassendiagramm: Stellt die Klassen eines Systems und deren Beziehungen dar.
  • Objektdiagramm: Zeigt Instanzen der Klassen zu einem bestimmten Zeitpunkt.
  • Komponentendiagramm: Stellt die physische Struktur von Software-Elementen dar.
  • Verteilungsdiagramm: Zeigt die physische Anordnung von Software und Hardware.

2. Verhaltensdiagramme

Verhaltensdiagramme zeigen das Verhalten eines Systems im Laufe der Zeit. Einige Beispiele für Verhaltensdiagramme sind:

  • Anwendungsfalldiagramm (Use Case Diagram): Stellt die Interaktionen zwischen Benutzern und Systemen dar.
  • Aktivitätsdiagramm: Beschreibt die Abläufe und Aktionen innerhalb eines Prozesses.
  • Zustandsdiagramm: Zeigt die Zustände eines Objekts und die Übergänge zwischen diesen Zuständen.

3. Interaktionsdiagramme

Interaktionsdiagramme sind eine Unterkategorie der Verhaltensdiagramme und konzentrieren sich auf die Interaktionen zwischen Objekten. Beispiele hierfür sind:

  • Sequenzdiagramm: Zeigt die zeitliche Reihenfolge der Nachrichten zwischen Objekten.
  • Kommunikationsdiagramm: Stellt die Interaktionen zwischen Objekten in einem Netzwerk dar.

Generalisierung, Spezialisierung, Polymorphie und Vererbung

1. Generalisierung

Generalisierung ist der Prozess, bei dem gemeinsame Merkmale und Eigenschaften mehrerer Klassen in einer übergeordneten Klasse zusammengefasst werden. Dies fördert die Wiederverwendbarkeit und reduziert die Redundanz im Code.


    public class Fahrzeug
    {
        public int Geschwindigkeit { get; set; }
        
        public void Fahren()
        {
            // Logik für das Fahren
        }
    }

2. Spezialisierung

Spezialisierung ist der Prozess, bei dem eine Unterklasse spezifische Eigenschaften und Methoden der übergeordneten Klasse erbt und zusätzlich eigene spezifische Merkmale hinzufügt. Dies ermöglicht die Anpassung und Erweiterung von Funktionalitäten.


    public class Auto : Fahrzeug
    {
        public int AnzahlTüren { get; set; }
        
        public void Hupen()
        {
            // Logik für das Hupen
        }
    }

3. Polymorphie

Polymorphie ermöglicht es, dass eine Methode in verschiedenen Klassen unterschiedlich implementiert werden kann, während die Methode denselben Namen hat. Dies wird häufig in Verbindung mit Vererbung verwendet, um spezifische Implementierungen in Unterklassen zu ermöglichen.


    public class Fahrrad : Fahrzeug
    {
        public override void Fahren()
        {
            // Logik für das Fahren mit einem Fahrrad
        }
    }

4. Vererbung

Vererbung ist ein zentrales Konzept in der objektorientierten Programmierung, bei dem eine Klasse (Unterklasse) von einer anderen Klasse (Oberklasse) erbt. Die Unterklasse erhält alle Attribute und Methoden der Oberklasse und kann diese erweitern oder überschreiben.


    public class Lastwagen : Fahrzeug
    {
        public int LaderaumVolumen { get; set; }
    }

Accessor

Accessor sind spezielle Methoden, die den Zugriff auf private oder geschützte Attribute einer Klasse ermöglichen. Sie bieten eine kontrollierte Schnittstelle zum Lesen (Getter) oder Schreiben (Setter) der Werte dieser Attribute. Dies fördert die Datenkapselung und schützt die Integrität der Daten.

Beispiel für Accessoren:


    public class Person
    {
        private string name;

        // Getter für das Attribut 'name'
        public string Name
        {
            get { return name; }
            set { name = value; }
        }
    }
